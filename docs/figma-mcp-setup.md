# Figma MCP 连接说明

Figma MCP 是设计实现工具链，不是 App 的运行依赖。具体安装方式取决于所使用的 Codex/IDE 客户端，应优先遵循该客户端的官方连接器或 MCP 配置文档，不在仓库中记录个人配置目录、访问令牌或机器专用路径。

## 最小能力

完成可靠的设计还原需要能够：

- 读取精确节点的设计上下文。
- 获取节点截图。
- 读取变量与组件属性。
- 获取设计元数据并在上下文过大时分段读取。
- 下载或导出设计实际使用的图片和 SVG。

## 安全配置

1. 通过客户端的凭据存储、OAuth 或本机环境变量提供 Figma 凭据。
2. 不把 Token 写入 `.env.example`、文档、截图、命令历史或 Git。
3. 只给完成设计读取所需的最小权限。
4. 连接后使用身份查询或读取一个已授权文件来验证权限。
5. 如果组织有数据区域要求，按组织策略配置区域，不在项目中硬编码个人设置。

## 项目内验证

连接成功后，对一个目标 Frame 依次获取：

1. Design context。
2. Screenshot。
3. Variable definitions。
4. 需要的图片或 SVG。

随后按 `docs/figma-handoff.md` 的 tokens → UI → Feature → route 流程实现。Figma 生成的代码只作为参考，最终代码必须符合当前 React Native、Expo Router 和 Monorepo 边界。

## 常见问题

- 只有技能说明、没有可调用的 Figma 工具：说明客户端尚未连接 MCP/插件，需在客户端完成连接并重启会话。
- 能读取文件但不能读取目标节点：检查 URL 是否指向精确节点以及账号是否有文件权限。
- 设计上下文过大：先读取元数据，再按节点分段获取，不要猜测缺失部分。
- 缺少图片：重新导出实际资源；不要用占位图冒充视觉一致。
- 连接失败：检查组织网络、OAuth 状态和客户端日志，但不要把 Token 粘贴到 Issue 或仓库文件。
